5️⃣ Multi‑Stage Docker Build

Below is a complete, production‑grade Dockerfile that builds a TorchServe‑based inference container, injects the model via a CSI driver, validates checksums, and hardens the image (CIS Docker Benchmark). 

# --------------------------------------------------------------
# Stage 0 – Builder (PyTorch + TorchServe)
# --------------------------------------------------------------
FROM nvcr.io/nvidia/pytorch:23.05-py3 AS builder

# 1️⃣ Install TorchServe & dependencies (pinned versions)
RUN pip install --no-cache-dir \
    torch==2.2.0 \
    torchvision==0.17.0 \
    torchserve==0.9.2 \
    torch-model-archiver==0.9.2 \
    transformers==4.40.2 \
    onnxruntime-gpu==1.18.0

# 2️⃣ Create a non‑root user (CIS‑Docker #5.5)
ARG USERNAME=appuser
ARG UID=10001
RUN adduser \
    --disabled-password \
    --gecos "" \
    --home "/home/${USERNAME}" \
    --shell "/sbin/nologin" \
    --uid "${UID}" \
    "${USERNAME}"

# --------------------------------------------------------------
# Stage 1 – Runtime (minimal, hardened)
# --------------------------------------------------------------
FROM nvidia/cuda:12.2.2-runtime-ubuntu22.04 AS runtime

# 1️⃣ Install runtime deps only (no compilers)
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
        ca-certificates \
        curl \
        libgomp1 \
        libjemalloc2 && \
    rm -rf /var/lib/apt/lists/*

# 2️⃣ Copy TorchServe binaries from builder
COPY --from=builder /usr/local/bin/torchserve /usr/local/bin/
COPY --from=builder /usr/local/bin/torch-model-archiver /usr/local/bin/
COPY --from=builder /usr/local/lib/python3.10/dist-packages /opt/torchserve/python/

# 3️⃣ Add non‑root user (same UID/GID as builder)
ARG USERNAME=appuser
ARG UID=10001
RUN adduser \
    --disabled-password \
    --gecos "" \
    --home "/home/${USERNAME}" \
    --shell "/sbin/nologin" \
    --uid "${UID}" \
    "${USERNAME}"

# 4️⃣ Create directories with proper permissions
RUN mkdir -p /home/${USERNAME}/model-store && \
    chown -R ${USERNAME}:${USERNAME} /home/${USERNAME}

# 5️⃣ Switch to non‑root user
USER ${USERNAME}
WORKDIR /home/${USERNAME}

# --------------------------------------------------------------
# Stage 2 – Model injection (CSI driver mounts model at /model)
# --------------------------------------------------------------
# The model is **not** baked into the image. It will be mounted
# at runtime via a CSI volume (see k8s manifest). The entrypoint
# validates the checksum before starting TorchServe.

COPY entrypoint.sh /usr/local/bin/entrypoint.sh
RUN chmod +x /usr/local/bin/entrypoint.sh

ENTRYPOINT ["/usr/local/bin/entrypoint.sh"]

entrypoint.sh (runtime verification)

bash

!/usr/bin/env bash

set -euo pipefail

MODEL_DIR="/model" CHECKSUM_FILE="${MODEL_DIR}/checksum.txt" EXPECTED=$(cat "${CHECKSUM_FILE}")

Compute checksum of the checkpoint file (assumes single .ckpt)

ACTUAL=$(sha256sum "${MODEL_DIR}"/*.ckpt | awk '{print $1}')

if [[ "${EXPECTED}" != "${ACTUAL}" ]]; then echo "❌ Model checksum mismatch! Expected ${EXPECTED}, got ${ACTUAL}" exit 1 fi

Archive model for TorchServe

torch-model-archiver \ --model-name llm \ --version 1.0 \ --serialized-file "${MODEL_DIR}"/*.ckpt \ --handler transformers_handler.py \ --extra-files "${MODEL_DIR}/config.json,${MODEL
