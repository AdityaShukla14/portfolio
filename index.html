1️⃣ Why a DevOps View Matters

Concern DevOps Impact Typical Failure Mode
Compute footprint Determines node‑type, autoscaling, spot‑vs‑on‑demand mix OOM, GPU under‑utilisation
Model lifecycle Versioned artifacts, reproducible builds Model‑drift, “unknown” checkpoint
Observability Latency spikes, GPU saturation, request‑level tracing SLA breach, silent failures
Security Secrets leakage, unauthorized inference Data‑exfiltration, compliance fines
Cost GPU hours dominate spend Budget overruns, idle nodes
Resilience Multi‑AZ, backup, graceful degradation Outage, loss of checkpoints
Governance Audit trails, data residency Regulatory penalties

A disciplined DevOps approach makes each of these predictable and automatable.

2️⃣ Core Architectural Parameters

# Parameter Why It Matters for DevOps Reference Implementation
1 Model Size & Precision (e.g., 124 M params, FP16 vs INT8) Drives GPU memory, batch size, latency vs throughput trade‑offs. torch.compile(..., mode="max-autotune") + torch.quantization
2 Compute Platform (GPU = NVIDIA A100, V100; TPU; CPU) Determines node pool, pricing, driver stack, and container base image. Terraform google_container_node_pool with accelerator_type = "nvidia-tesla-a100"
3 Inference Mode (real‑time, micro‑batch, async) Affects request routing, batching config, HPA thresholds. Triton --model-batch-size 8 + KEDA scaling on queue depth
4 Container Image (multi‑stage, GPU‑ready, SBOM, signed) Guarantees reproducibility, vulnerability control, runtime security. Dockerfile (see §3.2) → cosign sign
5 Orchestration & Autoscaling (K8s, KEDA, HPA, VPA) Guarantees elasticity, high‑availability, cost‑control. HorizontalPodAutoscaler on gpu_utilization metric
6 Model Artifact Store & Versioning (S3/GCS, MLflow, DVC) Immutable, traceable checkpoints; enables rollback. mlflow register-model + bucket lifecycle
7 Pre‑/Post‑Processing Pipelines (tokenizer, detokenizer, safety filters) Must be packaged, monitored, and can become bottlenecks. Prefect flow deployed as side‑car container
8 CI/CD Pipeline (IaC, image build, tests, promotion) End‑to‑end automation, gate‑keeping, canary rollouts. GitHub Actions + Argo Rollouts
9 Observability (metrics, logs, traces, GPU stats) Early detection of latency, OOM, drift. Prometheus + Grafana + Loki + OpenTelemetry + nvidia-dcgm-exporter
10 Security & Compliance (secrets, RBAC, network policies, image signing) Prevent data leakage, meet GDPR/HIPAA. Vault‑K8s auth, OPA Gatekeeper, Kyverno
11 Cost & FinOps (spot‑instance mix, cost‑per‑inference, budgets) Keeps spend predictable. Cloud Billing Export → Grafana panel
12 Resilience & DR (multi‑AZ, backup, chaos testing) Guarantees service continuity. Velero backups, Chaos‑Mesh experiments
13 Deployment Pattern (blue‑green, canary, A/B) Reduces risk of regressions. Argo Rollouts with metric analysis
14 Model Quality Monitoring (drift, hallucination, latency SLA) Detects degradation before it hits users. Evidently + custom drift detector
15 Governance & Auditing (audit logs, data‑residency tags) Satisfies internal & external policy. OPA policies + immutable S3 bucket versioning

3.1 Infrastructure as Code (Terraform)

# ──────────────────────────────────────────────────────────────
# 1️⃣ GKE Cluster (multi‑AZ, GPU node pool)
# ──────────────────────────────────────────────────────────────
provider "google" {
  project = var.gcp_project
  region  = var.gcp_region
}

resource "google_container_cluster" "llm_cluster" {
  name               = "llm-inference"
  location           = var.gcp_region
  remove_default_node_pool = true
  initial_node_count = 1

  # Enable Workload Identity for Vault ↔ K8s secret injection
  workload_identity_config {
    identity_namespace = "${var.gcp_project}.svc.id.goog"
  }
}

resource "google_container_node_pool" "gpu_pool" {
  name       = "gpu-pool"
  cluster    = google_container_cluster.llm_cluster.name
  location   = var.gcp_region
  node_count = var.gpu_node_count

  node_config {
    machine_type = "n1-standard-8"
    # Spot (preemptible) with on‑demand fallback
    preemptible  = true
    # Attach NVIDIA A100 GPU
    accelerators {
      type  = "nvidia-tesla-a100"
      count = 1
    }
    # Install NVIDIA driver automatically
    guest_accelerator {
      type  = "nvidia-tesla-a100"
      count = 1
    }
    # Enable GPU device plugin
    metadata = {
      disable-legacy-endpoints = "true"
    }
    oauth_scopes = [
      "https://www.googleapis.com/auth/cloud-platform"
    ]
    labels = {
      "gpu" = "a100"
    }
    taint {
      key    = "nvidia.com/gpu"
      value  = "present"
      effect = "NO_SCHEDULE"
    }
  }

  autoscaling {
    min_node_count = 1
    max_node_count = 8
  }
}

Key points

  • Node‑selector & taint – Guarantees only GPU‑enabled pods land on these nodes.
  • Preemptible (spot) mix – Reduces cost; fallback to on‑demand via node_pool autoscaling policy.
  • Workload Identity – Removes static service‑account keys; Vault injects secrets via K8s service accounts.

3.2 Multi‑Stage Docker Build & Image Hardening

# ──────────────────────────────────────────────────────────────
# 1️⃣ Build stage – compile model, create SBOM
# ──────────────────────────────────────────────────────────────
FROM nvidia/cuda:12.1.1-base-ubuntu22.04 AS builder

# Install OS deps
RUN apt-get update && apt-get install -y --no-install-recommends \
    python3-pip python3-venv git curl && \
    rm -rf /var/lib/apt/lists/*

# Create a non‑root user
ARG USERNAME=appuser
ARG UID=10001
RUN adduser --disabled-password --gecos "" --uid ${UID} ${USERNAME}

# Virtual env for reproducibility
ENV VENV=/opt/venv
RUN python3 -m venv ${VENV}
ENV PATH="${VENV}/bin:$PATH"

# Install Python deps (GPU‑enabled torch)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt \
    && pip install --no-cache-dir torch==2.2.0+cu121 -f https://download.pytorch.org/whl/torch_stable.html

# Pull model checkpoint (example: distilbert-base-uncased)
RUN pip install huggingface_hub && \
    python - <<'PY'
from huggingface_hub import snapshot_download
snapshot_download(repo_id="distilbert-base-uncased", local_dir="/model")
PY

# Generate SBOM (CycloneDX) and sign it
RUN pip install cyclonedx-bom && \
    cyclonedx-bom -r . -o /sbom/cyclonedx.json

# ──────────────────────────────────────────────────────────────
# 2️⃣ Runtime stage – minimal, GPU‑ready, signed, non‑root
# ──────────────────────────────────────────────────────────────
FROM nvidia/cuda:12.1.1-runtime-ubuntu22.04 AS runtime

# Install runtime deps only
RUN apt-get update && apt-get install -y --no-install-recommends \
    python3-venv && \
    rm -rf /var/lib/apt/lists/*

# Copy venv and model from builder
COPY --from=builder /opt/venv /opt/venv
COPY --from=builder /model /app/model
COPY --from=builder /sbom /app/sbom

ENV PATH="/opt/venv/bin:$PATH"
WORKDIR /app

# Add non‑root user (same UID/GID as builder)
ARG USERNAME=appuser
ARG UID=10001
RUN adduser --disabled-password --gecos "" --uid ${UID} ${USERNAME}
USER ${USERNAME}

# Runtime entrypoint (Triton or TorchServe wrapper)
COPY entrypoint.sh .
ENTRYPOINT ["./entrypoint.sh"]

# ----------------------------------------------------------------
# Image hardening metadata
# ----------------------------------------------------------------
LABEL org.opencontainers.image.source="https://github.com/yourorg/llm-inference"
LABEL org.opencontainers.image.created="${BUILD_DATE}"
LABEL org.opencontainers.image.version="${VERSION}"
LABEL org.opencontainers.image.licenses="Apache-2.0"
LABEL org.opencontainers.image.title="llm‑inference‑gpu"
LABEL org.opencontainers.image.base.name="nvidia/cuda:12.1.1-runtime-ubuntu22.04"

# Add seccomp & AppArmor profiles (mounted at runtime)
# (see ./security/ directory for the actual JSON/profile files)

What the Dockerfile solves

Requirement How it’s addressed
GPU‑ready base nvidia/cuda:12.1.1-runtime + torch compiled for CUDA 12.1
Multi‑stage Build‑time heavy deps are discarded → ~300 MB runtime image
SBOM & Signing cyclonedx-bom + cosign sign (CI step)
Non‑root appuser UID 10001, no root privileges
Runtime security Labels for seccomp/apparmor profiles; can be enforced by OPA/Kyverno
Deterministic env Virtualenv, pinned versions, reproducible requirements.txt
